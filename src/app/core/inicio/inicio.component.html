Reserva de canchas sintéticas
ADN Ceiba

Desarrollado por Oscar Alexander Ruiz Palacio, con el fin de poner en práctica los conocimientos aprendidos durante la Inducción de ADN Ceiba .

Se desarrolló un Backend en Java con Spring Boot y una arquitectura hexagonal o también conocida como el patron de puertos y adaptadores.

El frontend fue desarrollado con Angular haciendo uso de la arquitectura a 3 capas (core/shared/features).

Descripción del problema de negocio ADN:
  • Es posible el registro de canchas con la siguiente información: código, nombre, dirección, teléfono, tipo, y precio por reservar.
  • Es posible el registro de clientes con la siguiente información: nombre, correo electrónico, teléfono y número de cedula.
  • Es posible reservar canchas dado una fecha de reserva, una cancha y un cliente.
  • Las reservas no se puede realizar por un mismo cliente para una misma fecha en una misma cancha ya reservada anteriormente.
  • Las reservas no se puede realizar para el día actual.
  • Las reservas se realizan por el tiempo de una hora.
  • Si la reserva se realiza para los días viernes, sábados o domingos, se cobrará un 10% adicional al valor del precio por reservar.
